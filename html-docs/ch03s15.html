Copying Local Files

To duplicate an item in the working copy, remembering history, use the Copy (local) option ( Modify->Copy (local) (Alt+Q) ). Items can be copied from a source in the working copy to a destination in the working copy or they can be copied from the remote repository to the working copy. When copying between local paths, the copied item is copied (added) to the destination and is scheduled to be sent to the repository on the next commit operation. When copying from a remote repository to the working copy a checkout is performed and the item is scheduled to be added to the repository. The Copy dialog provides the following fields:

Source

The item path to be duplicated. The path may be a local path or remote URL.

Destination

The path to where the source will be copied. The path may only specify a remote URL.

Revision

Select the radio button and enter a value equal to a revision number. KEYWORDS are supported:

HEAD

The latest revision in the repository.

BASE

The "pristine" revision of an item in a working copy.

COMMITTED

The last revision in which an item changed before (or at) BASE.

PREV

The revision just before the last revision in which an item changed. (Technically, COMMITTED - 1.)

Date

Select the radio button and enter a value equal to a revision date or time. Specify date or time using any of the following syntax:

  • 2004-02-17

  • 15:30

  • 15:30:00.200000

  • 2004-02-17 15:30

  • 2004-02-17 15:30 +0230

  • 2004-02-17T15:30

  • 2004-02-17T15:30Z

  • 2004-02-17T15:30-04:00

  • 20040217T1530

  • 20040217T1530Z

  • 20040217T1530-0500

Message

A field with which to provide a free text message.
